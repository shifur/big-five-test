# ক্রিয়েট রিয়্যাক্ট অ্যাপ দিয়ে শুরু করা

এই প্রকল্পটি [ক্রিয়েট রিয়্যাক্ট অ্যাপ](https://github.com/facebook/create-react-app) দিয়ে বুটস্ট্র্যাপ করা হয়েছে।

## উপলব্ধ স্ক্রিপ্ট

প্রকল্প ডিরেক্টরিতে আপনি চালাতে পারবেন:

### `npm start`

ডেভেলপমেন্ট মোডে অ্যাপ চালায়.\
ব্রাউজারে দেখতে [http://localhost:3000](http://localhost:3000) খুলুন।

পৃষ্ঠাটি সম্পাদনা করলে রিলোড হবে.\
কনসোলে যেকোনো লিন্ট ত্রুটিও দেখতে পাবেন।

### `npm test`

ইন্টারেক্টিভ ওয়াচ মোডে টেস্ট রানার চালু করে.\
আরও তথ্যের জন্য [রানিং টেস্টস](https://facebook.github.io/create-react-app/docs/running-tests) অংশ দেখুন।

### `npm run build`

`build` ফোল্ডারে প্রোডাকশনের জন্য অ্যাপ বিল্ড করে.\
এটি প্রোডাকশন মোডে রিয়্যাক্টকে সঠিকভাবে বান্ডেল করে এবং সেরা পারফরম্যান্সের জন্য বিল্ড অপ্টিমাইজ করে।

বিল্ড মিনিফাই করা হয় এবং ফাইল নামগুলোতে হ্যাশ যুক্ত থাকে.\
আপনার অ্যাপ ডিপ্লয়ের জন্য প্রস্তুত!

আরও তথ্যের জন্য [ডিপ্লয়মেন্ট](https://facebook.github.io/create-react-app/docs/deployment) অংশ দেখুন।

### `npm run eject`

**দ্রষ্টব্য: এটি একমুখী প্রক্রিয়া। একবার `eject` করলে আর ফিরে যাওয়া যাবে না!**

বিল্ড টুল ও কনফিগারেশন পছন্দ না হলে যেকোনো সময় `eject` করতে পারেন. এই কমান্ড আপনার প্রকল্প থেকে একমাত্র বিল্ড ডিপেন্ডেন্সি সরিয়ে দেবে।

পরিবর্তে, এটি সব কনফিগারেশন ফাইল ও ট্রানজিটিভ ডিপেন্ডেন্সিগুলো (webpack, Babel, ESLint ইত্যাদি) আপনার প্রকল্পে কপি করবে যাতে আপনি সম্পূর্ণ নিয়ন্ত্রণ রাখতে পারেন. `eject` ব্যতীত সব কমান্ড তখনও কাজ করবে, তবে তারা কপি করা স্ক্রিপ্টগুলোর দিকে নির্দেশ করবে যাতে আপনি সেগুলো পরিবর্তন করতে পারেন. এই অবস্থায় আপনি নিজেই দায়ী।

`eject` ব্যবহার করার প্রয়োজন নেই. সীমিত ও মাঝারি ডিপ্লয়মেন্টের জন্য প্রদত্ত ফিচার সেট যথেষ্ট, তাই এটি ব্যবহার করতে বাধ্য বোধ করবেন না. তবে আপনি যখন প্রস্তুত, তখন কাস্টমাইজ না করতে পারলে এই টুল উপকারী হতো না—আমরা তা বুঝি।
